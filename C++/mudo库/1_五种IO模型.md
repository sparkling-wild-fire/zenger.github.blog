# 五种IO模型

IO 多路复用  一个线程多个socket   传统的是一个线程一个socket  

epoll通常注册的是非阻塞的socket =》 epoll返回一个可读的socket，你会不断地读缓冲区，但是我也不知道缓存区啥时候读完，如果是阻塞的话，读完最后一次会阻塞，那其他的socket全都会被阻塞了
（当然你也可知设置一个socket只读一次）

所以一般都是 非阻塞IO+多路复用+线程池

异步模型，结构体用一个指针，因为数据回来后，要写道这个指针的地址                 异步发送的传指针地址单纯为了减少参数大小嘛

ET的缺点：一直是某个socket在读数据，其他socket全被阻塞了
